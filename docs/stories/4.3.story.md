# S2C-3: Real-time Unlock for Paid Tiers

**Epic:** Sprint 2 Expandida — Parte C (Entrega do Valor Pago)
**Status:** Draft
**Type:** Backend + Frontend (API Route)
**Effort:** S (half day)
**Priority:** P1 (value differentiation)
**Depends on:** Auth system (existing), Tier system (existing)

## Story

Como usuario pago (Lite+), quero acessar dados em tempo real via SSE endpoint autenticado, para receber os dados que estou pagando sem delay.

## Acceptance Criteria

### AC-1: Authenticated SSE endpoint
- [ ] Route: `apps/web/src/app/api/v1/auth/stream/[channel]/route.ts`
- [ ] Validates user authentication via Supabase auth
- [ ] Validates user tier (Lite+ required for real-time)

### AC-2: Tier gating
- [ ] Free tier -> 403 with `{ error: "upgrade_required", upgrade_url: "/settings/billing" }`
- [ ] Lite+ -> stream access granted
- [ ] Reads from `cv:latest:{channel}` Redis keys (same as public, no delay)

### AC-3: SSE streaming
- [ ] Returns `text/event-stream` content type
- [ ] Polls Redis every 1s for latest data
- [ ] Sends SSE events with `data:` prefix
- [ ] Handles client disconnect gracefully

### AC-4: Tests
- [ ] Test: Free tier returns 403
- [ ] Test: Unauthenticated returns 401
- [ ] Test: Valid tier gets stream response headers

## Dev Notes

- Both public and auth streams read from same Redis keys for now
- Delay is enforced by cache TTL in publisher (15min for delayed cache)
- The real-time Railway SSE service is deferred to a separate PR (eng review decision)
- Valid channels: funding, liquidations, oi, trades, health
