**Epic:** Sprint 2 Expandida — Parte B (Ciclo de Venda)

## Story

Como CryptoVision em produção, preciso de Stripe configurado com products/prices, webhook completo com idempotency, e subscription lifecycle gerenciado, para cobrar assinaturas de forma segura e confiável.

## Acceptance Criteria

### AC-1: Stripe products and prices
- [ ] 4 products criados no Stripe: Lite ($14/mo), Starter ($29/mo), Pro ($99/mo), Enterprise ($299/mo)
- [ ] Cada product tem price mensal e anual (quando aplicável)
- [ ] Price IDs mapeados em constante `STRIPE_PRICES` (environment-specific)
- [ ] Free tier não tem product no Stripe (default)

### AC-2: Webhook handler completo
- [ ] `POST /api/webhooks/stripe` processa todos os eventos necessários
- [ ] Redis idempotency: `SET NX cv:stripe:event:{eventId} 1 EX 172800` (48h TTL)
- [ ] Se event ID já existe no Redis → return 200 (skip processing)
- [ ] `checkout.session.completed` → atualiza user plan + stripeCustomerId + stripeSubscriptionId
- [ ] `customer.subscription.updated` → atualiza user plan baseado no price → tier mapping
- [ ] `customer.subscription.deleted` → downgrade user para 'free'
- [ ] `invoice.payment_failed` → flag user (optional: send email notification)
- [ ] Logging de cada evento processado para debugging

### AC-3: Checkout session creation
- [ ] API route `POST /api/stripe/checkout` (authenticated)
- [ ] Creates Stripe checkout session with user metadata (userId)
- [ ] Success URL: `/dashboard?checkout=success`
- [ ] Cancel URL: `/pricing`
- [ ] Includes customer email from user record

### AC-4: Subscription management
- [ ] API route `POST /api/stripe/portal` (authenticated)
- [ ] Creates Stripe billing portal session for self-service management
- [ ] User can upgrade, downgrade, cancel via portal
- [ ] Portal redirect URL: `/settings/billing`

### AC-5: Tests
- [ ] Teste: Webhook rejects invalid signature → 400
- [ ] Teste: Webhook skips duplicate event (idempotency) → 200
- [ ] Teste: checkout.session.completed updates user plan
- [ ] Teste: subscription.deleted downgrades to free
- [ ] Teste: Checkout session created with correct metadata
- [ ] Teste: Portal session created for authenticated user
