# S2B-1: Auth Completo (Supabase Auth)

**Epic:** Sprint 2 Expandida — Parte B (Ciclo de Venda)
**Status:** Draft
**Type:** Full-Stack
**Effort:** M (1-2 dias)
**Priority:** P0 — Blocking
**Depends on:** S2A-0 (test framework)

## Story

Como visitante do CryptoVision, quero criar conta com email ou Google OAuth, fazer login, e ter minha sessão mantida, para acessar features autenticadas como dashboard e alertas.

## Acceptance Criteria

### AC-1: Signup page
- [ ] Route `/signup` com form: email + password + confirm password
- [ ] Google OAuth button ("Continue with Google")
- [ ] Validação client-side: email format, password min 8 chars, passwords match
- [ ] Após signup: email de verificação via Supabase Auth
- [ ] Redirect para `/signup/verify` com mensagem "Check your email"
- [ ] Referral code capture via URL param `?ref=CODE` → stored in user metadata

### AC-2: Login page
- [ ] Route `/login` com form: email + password
- [ ] Google OAuth button
- [ ] "Forgot password" link → `/forgot-password`
- [ ] Após login: redirect para `/dashboard` (ou URL anterior se veio de protected route)
- [ ] Error handling: invalid credentials, unverified email, rate limited

### AC-3: Session management
- [ ] Middleware refreshes Supabase session on every request (@supabase/ssr pattern)
- [ ] Session persists across page refreshes
- [ ] Logout button in navbar (when authenticated) → clears session → redirect to `/`
- [ ] Session expiry: redirect to `/login` with flash message

### AC-4: User record creation
- [ ] On first login: create/upsert user record in `users` table
- [ ] Fields: email, name (from OAuth profile), authId (Supabase auth.uid), plan='free'
- [ ] If referral code present: store in user metadata for later attribution

### AC-5: Tests
- [ ] Teste: Signup form validates email/password
- [ ] Teste: Login redirects to dashboard on success
- [ ] Teste: Middleware redirects unauthenticated from /dashboard
- [ ] Teste: User record created on first login
- [ ] Teste: Referral code captured from URL
