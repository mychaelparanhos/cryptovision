# S2B-3: Pricing Page + Checkout Flow

**Epic:** Sprint 2 Expandida — Parte B (Ciclo de Venda)
**Status:** Draft
**Type:** Frontend
**Effort:** M (1 dia)
**Priority:** P1
**Depends on:** S2B-2 (Stripe prices must exist)

## Story

Como visitante do CryptoVision, quero ver uma página de pricing clara com os planos disponíveis e poder iniciar o checkout com um clique, para escolher e pagar pelo plano que atende minhas necessidades.

## Acceptance Criteria

### AC-1: Pricing page
- [ ] Route `/pricing` com 5 tiers (Free, Lite, Starter, Pro, Enterprise)
- [ ] Card layout: name, price, features list, CTA button
- [ ] Starter ($29) highlighted como "Most Popular" (amber border)
- [ ] Toggle mensal/anual com desconto visível (ex: "$290/yr — Save 17%")
- [ ] Free tier: "Get Started" → signup
- [ ] Paid tiers: "Subscribe" → Stripe checkout (or signup first if not logged in)
- [ ] Enterprise: "Contact Us" → mailto or form

### AC-2: Checkout redirect
- [ ] Click "Subscribe" → if not authenticated, redirect to `/signup?redirect=/pricing&plan=starter`
- [ ] If authenticated → POST /api/stripe/checkout → redirect to Stripe checkout
- [ ] After successful checkout → redirect to `/dashboard?checkout=success` with success toast

### AC-3: Feature comparison
- [ ] Expandable feature comparison table below pricing cards
- [ ] Columns: feature name + checkmark per tier
- [ ] Features from TIER_LIMITS (API req/day, WS symbols, historical days, alerts)

### AC-4: Tests
- [ ] Teste: Pricing page renders 5 tier cards
- [ ] Teste: Monthly/annual toggle changes prices
- [ ] Teste: "Subscribe" redirects to signup when unauthenticated
- [ ] Teste: Starter tier has "Most Popular" highlight
