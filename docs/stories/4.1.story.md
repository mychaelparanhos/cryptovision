# S2C-1: Bybit Adapter

**Epic:** Sprint 2 Expandida — Parte C (Entrega do Valor Pago)
**Status:** Draft
**Type:** Backend (Ingestion Worker)
**Effort:** M (1 day)
**Priority:** P0 (core multi-exchange)
**Depends on:** Binance Adapter (existing)

## Story

Como trader de crypto futures, quero receber dados em tempo real da Bybit (trades, liquidations, funding, OI), para ter visao multi-exchange no CryptoVision.

## Acceptance Criteria

### AC-1: WebSocket connection
- [ ] BybitAdapter connects to `wss://stream.bybit.com/v5/public/linear`
- [ ] Subscribes to `publicTrade.{SYMBOL}` for each symbol
- [ ] Subscribes to `liquidation.{SYMBOL}` for each symbol
- [ ] Parses Bybit v5 WebSocket message format correctly

### AC-2: REST polling
- [ ] Polls funding rate from `GET /v5/market/tickers?category=linear&symbol={SYMBOL}`
- [ ] Polls open interest from same endpoint, extracts `openInterest` field
- [ ] Funding poll interval: 30min, OI poll interval: 15s

### AC-3: Event emission
- [ ] Emits `trade` events with correct MarketTrade shape
- [ ] Emits `liquidation` events with correct LiquidationEvent shape
- [ ] Emits `funding` events with correct FundingSnapshot shape
- [ ] Emits `oi` events with correct OISnapshot shape

### AC-4: Reconnection
- [ ] Exponential backoff reconnection matching BinanceAdapter pattern
- [ ] Health status reporting via getHealth()

### AC-5: Integration
- [ ] Registered in orchestrator (apps/ingestion/src/index.ts)
- [ ] Exported from adapters barrel (apps/ingestion/src/adapters/index.ts)

### AC-6: Tests
- [ ] Unit test: WebSocket message parsing (trades, liquidations)
- [ ] Unit test: REST polling data transformation
- [ ] Unit test: Symbol format handling

## Dev Notes

- Bybit v5 uses same symbol format as Binance (BTCUSDT) for linear perpetuals
- WebSocket subscription is message-based (send subscribe JSON after connect), not URL-based
- Message format: `{ "topic": "publicTrade.BTCUSDT", "data": [...] }`
- Follow BinanceAdapter pattern exactly (extends EventEmitter, implements ExchangeAdapter)
