# S2C-2: OKX Adapter

**Epic:** Sprint 2 Expandida — Parte C (Entrega do Valor Pago)
**Status:** Draft
**Type:** Backend (Ingestion Worker)
**Effort:** M (1 day)
**Priority:** P0 (core multi-exchange)
**Depends on:** Binance Adapter (existing)

## Story

Como trader de crypto futures, quero receber dados em tempo real da OKX (trades, liquidations, funding, OI), para completar a cobertura das 3 maiores exchanges no CryptoVision.

## Acceptance Criteria

### AC-1: WebSocket connection
- [ ] OKXAdapter connects to `wss://ws.okx.com:8443/ws/v5/public`
- [ ] Subscribes to `trades` channel with instId format (BTC-USDT-SWAP)
- [ ] Subscribes to `liquidation-orders` channel with instType: SWAP
- [ ] Symbol normalization: BTCUSDT -> BTC-USDT-SWAP

### AC-2: REST polling
- [ ] Polls funding rate from `GET /api/v5/public/funding-rate?instId={SYMBOL}`
- [ ] Polls open interest from `GET /api/v5/public/open-interest?instId={SYMBOL}&instType=SWAP`
- [ ] Funding poll interval: 30min, OI poll interval: 15s

### AC-3: Event emission
- [ ] Emits `trade` events with correct MarketTrade shape
- [ ] Emits `liquidation` events with correct LiquidationEvent shape
- [ ] Emits `funding` events with correct FundingSnapshot shape
- [ ] Emits `oi` events with correct OISnapshot shape

### AC-4: Symbol normalization
- [ ] Maps BTCUSDT -> BTC-USDT-SWAP for OKX API calls
- [ ] Reverse maps OKX instId back to canonical format in emitted events

### AC-5: Reconnection
- [ ] Exponential backoff reconnection matching BinanceAdapter pattern
- [ ] Health status reporting via getHealth()

### AC-6: Integration
- [ ] Registered in orchestrator (apps/ingestion/src/index.ts)
- [ ] Exported from adapters barrel (apps/ingestion/src/adapters/index.ts)

### AC-7: Tests
- [ ] Unit test: Symbol normalization (both directions)
- [ ] Unit test: WebSocket message parsing
- [ ] Unit test: REST polling data transformation

## Dev Notes

- OKX uses "BTC-USDT-SWAP" format for perpetual futures
- WebSocket requires sending subscribe message after connect (not URL-based)
- Subscribe message: `{ "op": "subscribe", "args": [{ "channel": "trades", "instId": "BTC-USDT-SWAP" }] }`
- Response format differs significantly from Binance — careful parsing required
- Follow BinanceAdapter pattern exactly (extends EventEmitter, implements ExchangeAdapter)
