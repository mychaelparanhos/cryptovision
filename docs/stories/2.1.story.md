# S2A-1: Aggregated Liquidation Heatmap Big 3

**Epic:** Sprint 2 Expandida — Parte A (Free Tier Robusto)
**Status:** Ready for Review
**Type:** Frontend + Backend (Full-Stack)
**Effort:** L (2-3 dias)
**Priority:** P1
**Depends on:** S2A-0 (test framework + publisher aggregation)

## Story

Como trader de crypto futures visitando o CryptoVision, quero ver um heatmap de liquidações agregado das 3 maiores exchanges (Binance, Bybit, OKX), para entender rapidamente onde estão as concentrações de liquidações sem abrir 3 tabs do Coinglass.

## Acceptance Criteria

### AC-1: Heatmap Canvas component
- [ ] Componente `LiquidationHeatmap` renderiza em `<canvas>` (ADR-009: Canvas, não SVG/D3)
- [ ] Eixo X: preço (range dinâmico baseado nos dados)
- [ ] Eixo Y: tempo (últimas 24h, scrollable)
- [ ] Cor: escala cold-to-hot baseada em valueUsd (#22C55E verde low → #F59E0B amber mid → #EF4444 vermelho high)
- [ ] Tooltip on hover: mostra exchange, symbol, side (LONG/SHORT), quantity, valueUsd, timestamp
- [ ] Responsive: adapta a largura do container

### AC-2: Aggregated data display
- [ ] Heatmap mostra dados AGREGADOS de todas as exchanges disponíveis (inicialmente só Binance)
- [ ] Label indica "Big 3 Aggregated" com badge mostrando quais exchanges estão ativas
- [ ] Se só 1 exchange disponível, mostra "Binance Only" com badge informativa
- [ ] Toggle para filtrar por exchange individual (free tier: só aggregated; paid: per-exchange toggle)

### AC-3: Data source via Redis cache
- [ ] Heatmap consome dados de `cv:cache:liq:agg:{symbol}:latest` (aggregated liquidations)
- [ ] Fallback: se cache vazio, mostra empty state com mensagem "Collecting data... Check back in a few minutes"
- [ ] Dados são delayed (15 min) para free tier — label "Delayed 15min" visível
- [ ] API endpoint: `GET /api/v1/public/liquidations/heatmap?symbol=BTCUSDT&period=24h`

### AC-4: Landing page integration
- [ ] Heatmap integrado na landing page como seção "Live Liquidation Heatmap"
- [ ] Posicionado acima do screener teaser
- [ ] Symbol selector: dropdown com top 5 (BTC, ETH, SOL, BNB, XRP), default BTC
- [ ] Auto-refresh a cada 30s (free tier delay é handled by cache TTL)

### AC-5: Testes
- [ ] Teste: LiquidationHeatmap renderiza com mock data sem erros
- [ ] Teste: Heatmap empty state quando sem dados
- [ ] Teste: API endpoint retorna dados corretos do cache
- [ ] Teste: API endpoint retorna 200 com array vazio quando cache vazio
- [ ] Teste: Symbol selector muda o símbolo e refetch dados

## Dev Notes

**Design System (DESIGN.md):**
- Background: #09090B (bg) / #18181B (surface para widget card)
- Border: #27272A, border-radius 12px, padding 16px
- Font data: JetBrains Mono Regular 400, right-aligned para números
- Cores semantic: #22C55E (positive), #EF4444 (negative), #F59E0B (amber accent)
- Heatmap específico: Canvas-based, cold-to-hot color scale
- Motion: 150ms transitions, ease-out

**Canvas rendering approach:**
- Use `requestAnimationFrame` for smooth updates
- Bin liquidations into price/time grid cells
- Color interpolation: HSL space (hue from green→amber→red, saturation constant)
- Performance: max ~10K data points visible, beyond that aggregate bins
