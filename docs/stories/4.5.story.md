# S2C-5: API Keys + Rate Limiting

**Epic:** Sprint 2 Expandida — Parte C (Entrega do Valor Pago)
**Status:** Draft
**Type:** Full Stack (API + Dashboard)
**Effort:** M (1 day)
**Priority:** P1 (API monetization)
**Depends on:** Auth system (existing), Tier system (existing), api_keys schema (existing)

## Story

Como usuario Starter+, quero gerar API keys para acessar dados do CryptoVision programaticamente, com rate limiting baseado no meu plano.

## Acceptance Criteria

### AC-1: API Key generation
- [ ] Generate random 32-byte hex key with `cv_` prefix
- [ ] Show full key once on creation (never stored in plain)
- [ ] Store SHA-256 hash + 8-char prefix in api_keys table
- [ ] Support naming keys for organization

### AC-2: API Key management page
- [ ] Dashboard page: `apps/web/src/app/(dashboard)/api-keys/page.tsx`
- [ ] List existing keys (prefix, name, usage, created date)
- [ ] Create new key with modal showing key once
- [ ] Revoke key (sets revokedAt timestamp)
- [ ] Dark mode design: #09090B, #18181B, #27272A, #F59E0B amber

### AC-3: API Key auth middleware
- [ ] Checks `Authorization: Bearer cv_...` header
- [ ] Hashes received key with SHA-256, looks up in api_keys table
- [ ] Rejects revoked keys
- [ ] Returns user info + tier for downstream use

### AC-4: Rate limiting
- [ ] Rate limit per API key based on user's tier (TIER_LIMITS.apiReqPerDay)
- [ ] Increment usageToday counter per request
- [ ] Return rate limit headers: X-RateLimit-Limit, X-RateLimit-Remaining

### AC-5: API routes
- [ ] POST `/api/v1/auth/api-keys` — create key
- [ ] GET `/api/v1/auth/api-keys` — list keys
- [ ] DELETE `/api/v1/auth/api-keys/[id]` — revoke key

### AC-6: Tests
- [ ] Test: Key generation produces correct format (cv_ prefix, 32-byte hex)
- [ ] Test: SHA-256 hashing is consistent
- [ ] Test: Revoked keys are rejected
- [ ] Test: Rate limiting enforced per tier

## Dev Notes

- api_keys table already exists in schema with keyHash, keyPrefix, usageToday fields
- Existing rate-limit.ts uses @upstash/ratelimit — extend for API key auth
- Key format: `cv_` + 64-char hex (32 bytes)
- Design matches existing dashboard pages (dark mode, amber accents)
